许多C语言的编译器提供了一种能力，允许在命令行中定义符号，用于启动编译过程。列如：当我们根据同一个源文件要编译出不同的一个程序的不同版本的时候，这个特性有点用处。(假定某个程序声明了一个某个长度的数组，如果机械内存有限，我们需要一个很小的数组，但是另外一个机械内存大一些，我们需要一个大一些的数组)

如下
	int arr[SZ] = {0};
	int i = 0;
	for(i=0;i<SZ;i++)
	{
		arr[i] = i;
	}
	for(i=0;i<SZ;i++)
	{
		printf("arr[%d] = %d\n", i, arr[i]);
	}
因为没有声明 SZ，所以会报错
但是编译前许多C语言的编译器可以用命令行定义符号，以便于在预编译阶段，如下是 gcc 编译器环境( 假设源文件test.c )
gcc test.c -D sz=5
输入之后便能顺利执行，输出
arr[0] = 0
arr[1] = 1
arr[2] = 2
arr[3] = 3
arr[4] = 4
